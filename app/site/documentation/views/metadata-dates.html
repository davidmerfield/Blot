Metadata / Dates

A blog post’s publish date is the same as the file’s creation date by default. You can specify a different date inside the file or as part of the file's path. Blot will schedule a blog post for future publication if you specify a date in the future.

Inside the file. The publish date is one of the properties of a blog post you can modify in the metadata at the start of a file. For example, this blog post will have a publish date of {{#date}}MMMM Do, YYYY{{/date}}:


Date: {{#date}}MMMM Do, YYYY{{/date}}

Blot supports a variety of date formats. The following all produce a blog post with the same date:

{{#date}}MM/DD/YYYY{{/date}}   {{#date}}MMMM D, YYYY{{/date}}
{{#date}}M.D.YYYY{{/date}}     {{#date}}YYYY-MM-DD 00:00{{/date}}

Using the file’s path

In the file's path. You can modify a post’s publish date using the file’s name or path. For example, all four of these posts share the same publish date:

 {{#date}}YYYY{{/date}} /    {{#date}}MM{{/date}} /    {{#date}}DD{{/date}} /    Post.txt
 {{#date}}YYYY{{/date}} /    {{#date}}MM{{/date}} /    {{#date}}DD-[Post.txt]{{/date}}
 {{#date}}YYYY{{/date}} /    {{#date}}MM-DD-[Post.txt]{{/date}}
 {{#date}}YYYY-MM-DD-00:00[-Post.txt]{{/date}}

You can use underscores, dashes and dots to seperate the components of the date. Please note that a date specified in the file’s metadata takes precedence over one specified by its path.
