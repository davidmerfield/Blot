Getting started with Blot

Blot turns files – text, images, documents, bookmarks – into blog posts automatically. When you add, remove or edit a file inside the folder, Blot publishes these changes to your blog. For more information about publishing, see Blog posts

Blot exists so that you can use your favorite creative tools. Since Blot's interface is just files and folders, you can use your favourite writing tools to blog.

You can organize your blog’s folder however you like. You can synchronize your folder using Dropbox or Git. All of these files become blog posts automatically when you put them somewhere inside your blog’s folder:

Formatting your blog posts

Blot supports a variety of standard tools to add formatting to your blog posts. You can use Markdown to add links, headings, lists and much more. Blot also renders mathematics set in \(\TeX\) and provides syntax highlighting for code snippets.

Read the formatting guide 
# Formatting

You can use *Markdown*, a lightweight markup language for plain text files.

Use it for links, headings, lists and much more.

Drafts

Files in a folder called ‘Drafts’ are not published. Instead, Blot creates a preview file which updates as you make edits.

Pages

Files in a folder called ‘Pages’ become pages instead of blog posts. They are added to your blog’s menu.

Metadata

Each blog post’s permalink and date is generated automatically from the file’s contents and location. You can override this {{#link}}metadata{{/link}} at the start of a file.

{{#link}}How to change a post’s date{{/link}} {{#link}} How to add tags to a post {{/link}} {{#link}} View the metadata reference {{/link}} 
Date: {{#date}}MMMM Do, YYYY{{/date}}
Permalink: /metadata

Metadata must start on the file’s first line and must be separated from the rest of the post by at least one blank line. You must leave a space after the colon.

Public files

Every file in your blog’s folder is public. Use this feature to distribute files to your readers and embed media in blog posts.

{{#link}}Guide to public files{{/link}}

Ignoring files

Blot ignores files and folders whose name starts with an underscore. Ignored files are public but don’t become blog posts.
